In this type of projects we need to:
1) provide an ability to build library with debug symbols
2) provide an ability to sign and install into GAC
3) provide .pc file for monodevelop "References"->"Packages" tab
4) provide an ability to create a local .nupkg (which may require writing .nuspec file)
5) perform AOT-compilation

which part of Makefile.include installs the .pc files?
not everything uses that Makefile.include
that is what MonoDevelop auto-generate
with the LINUX_PKGCONFIG var
so patching the path in that include file should do the trick
